Meyakara Design System — Colour

Colour Roles

Meyakara has no fixed brand colour. The foundation is constant. Collections activate on top of it via four accent pointer vars. Currently one collection is active: Dhundh.

01 — The four layers
Layer CSS vars Role Changes per collection?
Foundation --paper · --ink families Canvas and marks. The raw values never change. Dark modes (Dhundh before-dawn) re-point --bg and --fg away from paper/ink — but --paper and --ink themselves stay fixed underneath. Raw values: never.
--bg / --fg pointers: yes in dark modes.
Atmosphere --accent-tint Ambient surface wash. The sky of the collection. Applied at low opacity on section backgrounds. Yes — per collection
Structure --accent-soft Borders, card dividers, mid-weight UI. Gives the collection its texture without demanding attention. Yes — per collection
Soul --accent The single defining colour. CTAs, interactive states, key labels. The colour that names this escape. Yes — per collection
Depth --accent-deep High contrast moments. The darkest variant. Used where the soul needs weight or reach. Yes — per collection
02 — The four accent pointer vars
Atmosphere --accent-tint

Large surface wash. Section backgrounds at low opacity. The ambient sky of the collection.

Default: var(--paper-warm)
Structure --accent-soft

Borders, card hairlines, supporting elements. Mid-weight collection texture.

Default: var(--ink-mute)
Soul --accent

The defining colour. CTAs, hover states, key emphasis. Never more than ~10% of any surface.

Default: var(--ink)
Depth --accent-deep

Dark variant. High contrast moments. Also used for small interactive text when soul lacks contrast.

Default: var(--ink)
03 — Activating Dhundh
/* Dhundh owns the full page — body carries the dawn class */ <body class="collection-dhundh"> /* Phase 01 — City night: apply before-dawn on opening sections */ <section class="collection-dhundh--before-dawn"> <header>…</header> /* inherits the dark world */ /* hero, mist accents */ </section> /* Phase 02 — Stone in-between: applied directly as bg, not a class */ <section style="background: var(--dhundh-stone)">…</section> /* Phase 03 — Mountain clarity: inherits .collection-dhundh from body */ <section>…</section> <footer>…</footer> /* inherits dawn world */ </body>
04 — Dhundh colour roles

The same seven palette vars serve all three phases. The CSS re-assigns their roles depending on which surface mode a section lives in. Soul colours are outlined.

Phase 03 — Mountain clarity
.collection-dhundh
Atmosphere --dhundh-fog → --accent-tint #D8DDD9 · Earned — final sections only
Structure --dhundh-blue-grey → --accent-soft #8FA0A8 · Cool morning on rock
Soul ◼ --dhundh-pine → --accent #4F5C53 · Pine on paper
Depth --dhundh-deep → --accent-deep #2D332E · Absolute dark
Phase 01 — City night
.collection-dhundh--before-dawn
Background --dhundh-night → --bg #090C0A · Must stay deep
Soul ◼ --dhundh-mist → --accent #9EAAA4 · Only luminous thing in the dark
Structure --dhundh-pine → --accent-soft #4F5C53 · Pine silhouette, felt not read
Depth --dhundh-deep → --accent-deep #2D332E · Absolute dark
Phase 02 — The in-between
Stone is not mapped to
an accent pointer.

It is applied directly as a section background — background: var(--dhundh-stone). The one surface that belongs to neither city night nor mountain clarity. One transition section per page. It is a bridge, not a destination.

Section bg --dhundh-stone · #6E6A60
Text on stone var(--paper) · #F1ECE0
