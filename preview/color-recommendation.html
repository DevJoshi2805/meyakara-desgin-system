Meyakara Design System — Colour

Colour Roles

Meyakara has no fixed brand colour. The foundation is constant. Collections activate on top of it. Every colour decision in this system follows from one rule: the escape begins where the lobby ends.

01 — The four layers
Layer CSS vars Role Changes per collection?
Foundation --paper · --ink families Canvas and marks. The brand at rest. Header, footer, checkout, meta pages. Never
Atmosphere --accent-tint Ambient surface wash. The sky of the collection. Applied at low opacity on section backgrounds. Yes — per collection
Structure --accent-soft Borders, card dividers, mid-weight UI. Gives the collection its texture without demanding attention. Yes — per collection
Soul --accent The single defining colour. CTAs, interactive states, key labels. Use sparingly. The colour that names this escape. Yes — per collection
Depth --accent-deep High contrast moments. The darkest variant. Used where the soul needs weight or reach. Yes — per collection
02 — The four accent pointer vars
Atmosphere --accent-tint

Large surface wash. Section backgrounds at low opacity. The ambient sky of the collection.

Default: var(--paper-warm)
Structure --accent-soft

Borders, card hairlines, supporting elements. Mid-weight collection texture.

Default: var(--ink-mute)
Soul --accent

The defining colour. CTAs, hover states, key emphasis. Never more than ~10% of any surface.

Default: var(--ink)
Depth --accent-deep

Dark variant. High contrast moments. Also used for small interactive text when the soul lacks contrast.

Default: var(--ink)
03 — The root default

The :root accent default is the foundation — not Lino, not any collection. No collection is active until a .collection-* class is applied to a wrapper element. A page without a collection class renders in paper and ink only. That is the correct behaviour — the escape has not been declared yet.

Lino is not the brand colour. It is the most accessible collection, used as the year-round default when a collection is active. It is activated explicitly via .collection-lino.

/* :root defaults — no collection active */ --accent: var(--ink); --accent-soft: var(--ink-mute); --accent-deep: var(--ink); --accent-tint: var(--paper-warm); /* Activate a collection on its wrapper */ <main class="collection-lino"> … </main> <main class="collection-dhundh"> … </main> <main class="collection-raat"> … </main> /* ← inverts foundation */ /* Brand chrome stays outside — always */ <header class="mk-lobby"> … </header> <footer class="mk-lobby"> … </footer>

05 — The lobby rule
The rule
Brand chrome never carries collection accent.

These surfaces belong to the brand, not to any collection. They stay in paper and ink regardless of which collection is active on the page.

.mk-lobby

Resets all --accent vars to foundation values. Apply to header, footer, drawers, checkout, account.

Site header
Navigation, wordmark, bag. Always paper. Always ink.
Site footer
Links, tagline, copyright. Always paper. Always ink.
Checkout & account
Transactional surfaces. No escape is active here.
Error & loading states
404, empty states, loading. Foundation only.
